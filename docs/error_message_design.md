# エラーメッセージ定義書

本ドキュメントでは、入力チェックルールおよび表示メッセージを定義します。

## 1. バリデーション共通方針

以下の原則に従って実装します。

- 即時フィードバック: 入力不備がある場合は、各入力項目の直下にエラーメッセージを表示する。
- 入力値の保持: エラー時も入力済みの内容は破棄せず、フォームに再表示（old ヘルパの利用）する。
- サーバーサイド実行: セキュリティ担保のため、必ずサーバー側で最終チェックを行う。

## 2. 項目別バリデーションルール

### 2.1 会員登録・認証系 (F-01, F-02)

安全なアカウント作成とログインのためのルールです。
| 項目 | 必須 | 型 / 文字数 | 制約 | エラーメッセージ例 |
| :------ | :- | :------------ | :-------------- | :-------------------- |
| 名前 | ◯ | 文字列 / 最大 255 文字 | - | 名前を入力してください。 |
| メールアドレス | ◯ | email 形式 | Unique（登録済み不可） | このメールアドレスは既に登録されています。 |
| パスワード | ◯ | 文字列 / 8 文字以上 | 確認用パスワードと一致すること | パスワードは 8 文字以上で入力してください。 |

### 2.2 記事管理系 (F-05, F-06)

ブログ投稿の品質を維持するためのルールです。
| 項目 | 必須 | 型 / 文字数 | 制約 | エラーメッセージ例 |
| :--- | :- | :------------ | :- | :----------- |
| タイトル | ◯ | 文字列 / 最大 255 文字 | - | タイトルは必須です。 |
| 本文 | ◯ | テキスト | - | 本文を入力してください。 |

## 3. システムエラーメッセージ定義

バリデーション以外の例外処理（非機能要件）に関する定義です。
| 状況 | HTTP ステータス | 表示メッセージ | 対応方針 |
| :------ | :-------- | :---------------------------- | :------------- |
| ページ未検出 | 404 | 指定された記事は見つかりませんでした。 | 専用の 404 ページへ誘導 |
| 権限エラー | 403 | 他人の記事を編集・削除することはできません。 | ダッシュボードへリダイレクト |
| サーバーエラー | 500 | システムエラーが発生しました。時間を置いてお試しください。 | ログを出力し、管理者へ通知 |
